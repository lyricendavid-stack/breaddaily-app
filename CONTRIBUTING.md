- Use the **Feature Request** template.

#### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the UI adheres to our **Design Rules** (No clutter, rounded corners, high readability).
4. Issue a PR with a clear description of changes.
